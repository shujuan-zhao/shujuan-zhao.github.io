# Contributing Guidelines

Thank you for your interest in contributing to this academic portfolio website!

## How to Contribute

While this is a personal academic website, I welcome:

1. **Bug Reports**: If you notice any issues with the site
2. **Typo Fixes**: Help improve the content quality
3. **Technical Improvements**: Suggestions for better performance or accessibility
4. **Feature Suggestions**: Ideas for new sections or functionality

## Reporting Issues

1. Check if the issue already exists in the [Issues](https://github.com/shujuan-zhao/shujuan-zhao.github.io/issues) section
2. Create a new issue with a clear, descriptive title
3. Include:
   - Steps to reproduce (for bugs)
   - Expected behavior
   - Actual behavior
   - Browser and OS information
   - Screenshots if applicable

## Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/improvement-name`)
3. Make your changes
4. Test locally with `bundle exec jekyll serve`
5. Commit with clear messages
6. Push to your fork
7. Submit a Pull Request

## Code Style

- Follow the existing code style
- Use meaningful variable and function names
- Add comments for complex logic
- Ensure accessibility standards are met

## Testing

Before submitting:
- Run `bundle exec jekyll build` without errors
- Test in multiple browsers
- Check responsive design
- Validate HTML/CSS

## Questions?

Feel free to open an issue for any questions about contributing.
